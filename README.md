# Brocs · Партнёрская сеть — как выложить в интернет

Это тот же код приложения, что был в чате, только теперь он не «артефакт Claude»,
а обычный сайт, который можно выложить на бесплатный хостинг и открывать с
любого телефона по ссылке.

Единственное отличие: раньше данные хранились в специальном хранилище Claude
(`window.storage`), которое работает только внутри чата. Здесь вместо него —
настоящая база данных (Supabase, бесплатно), чтобы владелец, менеджеры и
специалист видели одни и те же данные с разных телефонов.

Займёт 10–15 минут в первый раз. Дальше — просто ссылка, которая работает всегда.

---

## Шаг 1. База данных (Supabase) — 3 минуты

1. Зайдите на **https://supabase.com** → «Start your project» → зарегистрируйтесь
   (можно через Google или GitHub).
2. Создайте новый проект (**New project**). Придумайте пароль для базы данных
   — просто сохраните его куда-нибудь, дальше он не понадобится.
3. Подождите 1–2 минуты, пока проект создастся.
4. Слева в меню откройте **SQL Editor** → **New query**.
5. Откройте файл `supabase.sql` из этого архива, скопируйте всё содержимое,
   вставьте в редактор и нажмите **Run**.
6. Слева в меню откройте **Project Settings → API**. Там будут:
   - **Project URL** (например `https://abcdefgh.supabase.co`)
   - **anon public** ключ (длинная строка)

   Оба значения понадобятся в шаге 3.

---

## Шаг 2. Выложить код на GitHub — 3 минуты

1. Зайдите на **https://github.com** → зарегистрируйтесь, если ещё нет аккаунта.
2. Нажмите **New repository**. Назовите как угодно, например `brocs-app`.
   Оставьте его **Private**, если не хотите, чтобы код видели посторонние.
3. На странице нового репозитория нажмите **uploading an existing file**
   (или «Add file» → «Upload files»).
4. Перетащите туда **все файлы и папки из этого архива** (кроме `node_modules`
   и `dist` — их тут и нет) и нажмите **Commit changes**.

---

## Шаг 3. Хостинг (Vercel) — 3 минуты

1. Зайдите на **https://vercel.com** → **Sign Up** → войдите через GitHub
   (проще всего — тогда Vercel сразу увидит ваш репозиторий).
2. Нажмите **Add New… → Project**.
3. Найдите репозиторий `brocs-app` (или как вы его назвали) → **Import**.
   Vercel сам определит, что это Vite-проект — ничего менять не нужно.
4. Перед первым деплоем откройте **Environment Variables** и добавьте две
   переменные (значения — из шага 1, пункт 6):
   - `VITE_SUPABASE_URL` = ваш Project URL
   - `VITE_SUPABASE_ANON_KEY` = ваш anon-ключ
5. Нажмите **Deploy**. Подождите минуту — Vercel покажет готовую ссылку вида
   `https://brocs-app.vercel.app`.

Готово — эта ссылка открывается с любого телефона, и все данные общие для
всех, кто по ней заходит (владелец, менеджеры, специалист).

---

## Если что-то не так с переменными

Если после деплоя приложение открывается, но данные не сохраняются —
проверьте в Vercel: **Project → Settings → Environment Variables**, что оба
значения из Supabase вписаны без лишних пробелов, и нажмите **Redeploy**
(Deployments → ⋯ → Redeploy) после любого изменения переменных.

---

## Дальнейшие изменения

Если позже понадобится что-то поменять в приложении — присылайте код
`src/App.jsx` из этого проекта, я буду вносить правки в него же, а вы просто
заново загрузите обновлённый файл на GitHub (Vercel пересоберёт сайт
автоматически, за 1 минуту).
